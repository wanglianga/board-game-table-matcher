# 桌游吧 · 组局拼桌

## 原始需求

> 请制作桌游吧组局拼桌页面，Vue3 页面呈现游戏库、玩家人数、预计时长、规则复杂度、主持人、桌号、拼桌进度、教学需求和饮品套餐。玩家在手机上筛选推理、策略、派对或合作类游戏，看到已有成员水平、缺几人、开局倒计时、能否中途退出和主持人位置；店内屏幕滚动展示等待桌、已成局、临时换桌和即将讲规则的场次。页面可以热闹，但不能让游戏封面抢掉关键信息，玩家一眼要知道自己适不适合加入、还有多久开、到哪张桌集合。
>
> 扩展新手入局保护：玩家标记第一次玩、社恐、只想轻松局或不想长时长后，页面优先给出有主持教学、规则轻、气氛友好且允许观望的桌；若选择硬核桌，卡片要说明阅读规则耗时和退出影响。
> 扩展多桌候补冲突：同一玩家排了两个等待桌，又被朋友邀请第三桌时，页面根据预计成局时间、桌间距离和主持人准备情况提示冲突；确认其中一局后，其他候补名额释放并通知队伍还缺几人。

## 项目简介

桌游吧组局拼桌平台，包含两个视图：

- **玩家手机端** (`/`)：分类筛选、对局卡片、拼桌详情、加入/退出、饮品套餐
- **店内大屏端** (`/screen`)：等待桌滚动、已成局展示、即将讲解高亮、换桌通知

## 核心功能

### 1. 新手入局保护

玩家可在页面顶部标记个人偏好，系统会根据偏好智能推荐合适的桌台：

| 偏好标记 | 推荐逻辑 |
|---------|---------|
| 第一次玩 | 优先推荐有主持教学、规则轻、低复杂度的桌台，允许观望 |
| 社恐友好 | 优先推荐气氛友好、可先观望、规则轻的桌台 |
| 轻松局 | 优先推荐规则轻、气氛友好、短时长、可中途退出的桌台 |
| 不想长时长 | 优先推荐短时长（30分钟内）、可中途退出的桌台 |

**硬核桌提醒**：当玩家有任何偏好标记时，硬核桌会显示醒目的警告卡片，说明：
- 规则阅读所需时间
- 中途退出的影响
- 不允许中途退出等限制

### 2. 多桌候补冲突处理

当玩家已在 2 桌候补，尝试加入第 3 桌时，系统会触发智能冲突检测：

**冲突评估维度**：
- ⏰ **时间冲突**：两桌开局时间差小于 10 分钟
- 📍 **距离冲突**：两桌不在同一区域，且开局时间差小于 15 分钟
- 👨‍🏫 **主持人状态**：主持人未就位且开局时间差小于 20 分钟

**风险等级**：
- 🟢 低风险：时间和位置相对宽松
- 🟡 中风险：存在时间或距离冲突
- 🔴 高风险：多重冲突叠加

**朋友邀请标记**：朋友邀请的桌台会显示粉色邀请标记，冲突提示中也会给出针对性建议。

**候补释放机制**：确认选择某一桌后，其他候补名额自动释放，并通知被释放的桌台还缺几人。

## 技术栈

- Vue 3 + TypeScript + Vite
- Tailwind CSS 3
- Pinia 状态管理
- Vue Router 4
- Lucide Vue Next 图标

## 启动方式

### 前置要求

- Node.js 18+
- npm

### Docker 一键启动（推荐）

#### 前置要求

- Docker
- Docker Compose

#### 启动步骤

```bash
docker compose up --build
```

访问地址：http://localhost:8080

后台运行：

```bash
docker compose up --build -d
```

停止服务：

```bash
docker compose down
```

### 本地开发启动

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动服务

```bash
npm run dev
```

访问地址：http://localhost:5173

### 页面路由

| 路由 | 说明 |
|------|------|
| `/` | 玩家手机端 |
| `/screen` | 店内大屏端 |
